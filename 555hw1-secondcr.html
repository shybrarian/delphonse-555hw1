Second Criterion: User control and freedom

"Users often choose system functions by mistake and will need a clearly marked "emergency exit" to leave the unwanted state without having to go through an extended dialogue. Support undo and redo."

O NGA

When the user clicks on a record in the search results, it shows up in an overlay screen. It is easy to exit this screen because there is a big X to the left of the record, and ‘underneath’ the overlay the previous search results screen is still visible. Similarly, any external links open in a new tab, so the user is always only one quick away from the previous screen view or tab.(See Screenshot)


X Met

Records open on a new webpage, but within records, the user can click Next record or Previous record on the top right of the page to toggle through the search results list. However, the user must click the back button in order to return to the original search results page. This process could be tedious and confusing if the user has scrolled through many records. Thankfully, they can view search history if they get frustrated and click on the main search rather than hit the back button until they return to the results page.


O Getty

When you view a record in this catalog, it opens a new webpage. On this record page, the user can see the number of the result they are on (eg. Result 4) and toggle between results using the Previous/Next function. Instead of having to click the back button to return to the search results page, the user can simply select Back to results list. (See Screenshot)

Ringling

This catalog opens outside links in new tabs so the user doesn’t have to click the back button to exit. When the user clicks on a record, it goes to a new webpage, but the user can easily return to the search results by clicking Results list.


X MFA

The language used on the record webpages is a little unclear, but the user can return to the search results by selecting Titles. There is also a Previous/Next function to sift through records. One rather inconvenient aspect is that the user cannot search again under the same location (Museum of Fine Arts) from the results page. If the user makes a search from the search bar on a results page, they must hit the search button to go to the search results, then adjust the necessary refinements using the filters on the right hand panel.
